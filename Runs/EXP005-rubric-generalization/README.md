# EXP-005: Cross-Task Rubric Generalization Analysis

**Status:** Not applicable for medium benchmark — P3 Nice to Have
**Paper section:** Section 3 (Method), rubric criteria generalization claim
**Backlog entry:** EXP-005 in `Docs/paperv2/experiment_backlog.md`

---

## Purpose

Demonstrate that CoEval rubric criteria generalize meaningfully across task
types by analysing criterion overlap and semantic similarity between rubrics
generated for different tasks. This validates the methodological claim that
CoEval's Phase 2 rubric generation produces task-appropriate criteria that
nonetheless share a common quality vocabulary (accuracy, clarity, completeness)
while also producing task-specific discriminating criteria.

The backlog notes this is "purely analytical, no new experiments needed
— 30 min analysis."

---

## Hypothesis

Rubric criteria generated by CoEval Phase 2 will exhibit:
1. A set of universal criteria shared across all task types (accuracy,
   clarity, completeness, coherence) — reflecting general text quality.
2. A set of task-specific criteria that are semantically distinct
   across tasks (e.g. technical_accuracy for code, faithfulness for
   summarization, actionability for emails).

Semantic similarity between task rubrics (measured via sentence embeddings)
will be moderate (cosine similarity 0.5-0.7) for shared criteria and low
(< 0.3) for task-specific criteria, demonstrating appropriate specialization.

---

## Experimental Design

### Core analysis (0 API calls, ~30 minutes)

Analyse the rubric files from medium-benchmark-v1:
- `Runs/medium-benchmark/phase2_rubric/text_summarization.rubric.json`
- `Runs/medium-benchmark/phase2_rubric/code_explanation.rubric.json`
- `Runs/medium-benchmark/phase2_rubric/email_composition.rubric.json`
- `Runs/medium-benchmark/phase2_rubric/data_interpretation.rubric.json`

Steps:
1. Extract all (criterion_name, criterion_description) pairs per task
2. Compute pairwise cosine similarity between criterion descriptions using
   sentence-transformers (e.g. all-MiniLM-L6-v2 or all-mpnet-base-v2)
3. Cluster criteria by semantic similarity (DBSCAN or agglomerative clustering)
4. Identify shared clusters (present in 3+ tasks) vs. task-specific criteria
5. Visualise as a heatmap and dendrogram

### Optional extension (2 new tasks, ~$0.25 API cost)

Add two new domain tasks to test generalization further:
- `legal_document_analysis` (20 items, auto rubric)
- `medical_summarization` (20 items, auto rubric)

These tasks are defined in `config.yaml` with `rubric: auto`, so Phase 2
will generate new rubrics from scratch. The resulting rubrics are then
included in the overlap analysis to test whether clinical and legal quality
criteria share structure with general NLP task criteria.

## Core Analysis Commands

Run rubric overlap analysis against medium-benchmark-v1:
```
coeval analyze rubric-overlap \
    --run Runs/medium-benchmark \
    --out Runs/EXP005-rubric-generalization/reports
```

Or directly with Python:
```python
from analyzer.rubric_generalization import compute_rubric_overlap
results = compute_rubric_overlap(
    rubric_dirs=[
        "Runs/medium-benchmark/phase2_rubric",
        "Runs/EXP005-rubric-generalization/phase2_rubric",  # if extension run
    ],
    embedding_model="all-MiniLM-L6-v2",
    out_dir="Runs/EXP005-rubric-generalization/reports"
)
```

## Extension Task Commands

Only needed if running the optional new tasks:

coeval run --config Runs/EXP005-rubric-generalization/config.yaml
coeval analyze rubric-overlap \
