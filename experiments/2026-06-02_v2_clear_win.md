# EXP-V2 — Make CoEval a clear win (branch `v2`)

**Status:** in_progress · **Started:** 2026-06-02 · **Branch:** `v2`

## Hypothesis
CoEval currently *matches* a strong single judge (G-Eval) on clean-task point
correlation because (a) judges score on a coarse 3-level scale (High/Medium/Low →
1.0/0.5/0.0) that caps discriminative power, and (b) aggregation is an unweighted
mean. CoEval should pull clearly ahead when we (1) raise the scoring resolution,
(2) use reliability-weighted aggregation, and (3) measure it where single judges
structurally fail (ranking stability, family bias). Falsified if, after these
changes, CoEval still only matches the best single judge on every axis.

## Results (append as they land)
- **A (ranking stability on clean QA) — NULL.** Bootstrap B=2000 over the 191 QA
  datapoints: every single judge AND the ensemble recover the true ranking on
  ~100% of resamples (full=0.999–1.000, top2=0.999–1.000). On an objective task
  with well-separated models (0.969 / 0.942 / 0.832) a single judge is already
  perfectly stable, so there is no ensemble advantage to show here. This matches
  the Figure-2 lesson: clean QA is a validity floor, not where the ensemble wins.
  → Pivot: the ensemble advantage must be measured where single judges are
  noisy/biased/disagree (subjective tasks, close models, bias present), and via
  the finer scale (improves CoEval's own resolution). `reports/v2_ranking_stability.json`.

- **B+D (judge-choice regret + reliability-weighted aggregation) — WIN.** Summarization
  vs BERTScore, n=600. Single-judge correlation ranges from **−0.006 (gpt-3.5-turbo,
  anti-correlated) to +0.249 (gpt-4o-mini)**: judge-choice regret = **0.255**, and the
  best judge is task-dependent (claude-haiku is best on QA-adjacent runs, gpt-4o-mini
  here), so a single judge cannot be chosen reliably a priori. The cross-family
  ensemble is the low-regret choice (never anti-correlated). Unsupervised
  **reliability-weighted aggregation = 0.242 > plain mean 0.227**, recovering most of
  the gap to the best single judge by down-weighting the low-agreement judge, with NO
  ground truth used. `reports/v2_aggregation_regret.json`.
  → Adopt reliability weighting in the aggregator; lead the paper's ensemble argument
  with judge-choice regret (a single judge can be worse than useless; the ensemble
  cannot), not with point-accuracy.

- **Self-preference flip (controlled diff-in-diff) — WIN.** Verticals include
  gpt-4o-mini as both judge and student. On clinical_reasoning the gpt-4o-mini judge
  scores itself 0.856 > gpt-3.5-turbo 0.841, while the cross-family judges score it
  0.867 < gpt-3.5-turbo 0.889. Self-preference = delta_same - delta_cross = **+0.038**,
  enough to **flip the #1 model** (same-family judge ranks gpt-4o-mini first;
  cross-family ranks gpt-3.5-turbo first). Legal is a control (-0.041, no flip; the gap
  there is too large to flip). A single same-family judge yields a self-serving ranking;
  the vendor-disjoint ensemble removes it. `EXP006/reports/v2_self_preference.json`.
  → Promote §5.4 from a structural argument to a measured ranking-flip result.

## Conclusion
- (in progress) The win is not point-accuracy on clean tasks. Refocus on:
  (i) finer judge scale to raise resolution; (ii) judge-choice regret on
  subjective/close tasks; (iii) bias-amplification and contamination stress tests
  where single judges structurally fail. Honesty note: do not claim a stability
  win on clean QA — there is none.
